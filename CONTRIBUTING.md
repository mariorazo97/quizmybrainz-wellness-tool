# Contributing to Body & Mind Detective

First off, thank you for considering contributing to Body & Mind Detective! It's people like you who make this tool better for educators, students, and mental health professionals worldwide.

### Reporting Bugs

Before creating bug reports, please check the existing issues to avoid duplicates. When creating a bug report, include:

- **Clear, descriptive title**
- **Steps to reproduce** the behavior
- **Expected behavior** vs what actually happened
- **Screenshots** if applicable
- **Browser and OS information**

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion:

- **Use a clear, descriptive title**
- **Provide detailed description** of the suggested enhancement
- **Explain why this enhancement would be useful** to most users
- **List any similar features** in other tools, if applicable

### Pull Requests

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Test thoroughly across different browsers
4. Ensure your code follows the existing style
5. Update documentation if needed
6. Write a clear commit message

## ğŸ“‹ Development Guidelines

### Code Style

- **HTML**: Use semantic HTML5 elements, proper indentation (2 spaces)
- **CSS**: Follow the existing naming conventions, use CSS variables for colors
- **JavaScript**: Use vanilla JS, clear variable names, add comments for complex logic
- **Accessibility**: Maintain WCAG 2.1 AA standards

### Testing Checklist

- [ ] Test on Chrome, Firefox, Safari, Edge
- [ ] Test on mobile devices (iOS and Android)
- [ ] Verify all links work correctly
- [ ] Check that animations are smooth
- [ ] Ensure text is readable (contrast ratios)
- [ ] Test with screen readers if modifying accessibility features
